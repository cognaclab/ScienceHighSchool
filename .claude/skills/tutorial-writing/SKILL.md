# Writing tutorials and slides in this repo's style

The audience is first-year Japanese high-school students (15–16 y.o., little
or no programming experience, English as a second language). For COSMOS-level
material, keep the same structure but raise the maths and drop the language
scaffolding.

## Document conventions (tutorials)

Copy the YAML header from an existing tutorial (`format: html`,
`embed-resources: true`, `toc: true`, theme `cosmo`). Then follow this
skeleton, in order:

1. **Word list callout** — table of key English terms with Japanese glosses
   (単語リスト). 5–7 terms max, only ones actually used.
2. **Hook** — a concrete, familiar situation (ramen queue, slot machine,
   TikTok trend) before any formalism.
3. **Small steps** — one new idea per section. Every code chunk must be
   short (< 15 lines), runnable top-to-bottom in a fresh session, and produce
   visible output (a value or a plot). Set `set.seed()` where reproducibility
   matters for the prose.
4. **"Try it!" callouts** (`.callout-tip`) — exercises that modify code the
   student just ran. Always ask them to *predict before running*.
5. **"The big idea" callout** (`.callout-important`) — one per tutorial,
   connecting the toy model to real research.
6. **"What you learned"** — 3–5 bullets, then a one-line cliffhanger bridge
   to the next tutorial.

## Language rules

- Short sentences. One clause where possible. No idioms.
- Define every technical term at first use; add the Japanese gloss in
  parentheses for load-bearing terms（例：確率）.
- Greek letters: always name them ("$\alpha$ (alpha)") and say what they
  *do*, not just what they are.
- Talk to the student as "you"; the tone is warm and curious, never cute.

## Code rules

- Base R + ggplot2 only. No tidyverse verbs (no `%>%`, `mutate`, `map`) —
  students haven't learned them and they add magic.
- Reusable model code lives in `R/bandit_functions.R`; tutorials load it
  with `source("R/bandit_functions.R")` (works because `_quarto.yml` sets
  `execute-dir: project`). Tutorial 3 is the exception: it *builds* the
  functions inline because that is the lesson.
- If you change `R/bandit_functions.R`, keep tutorial 3's inline copies
  identical, and update `R/verify_simulations.R` (see the
  `verify-simulations` skill).

## Slides (revealjs decks in slides/)

- One idea per slide; fragments (`.fragment`, `.incremental`) for reveals.
- Figures are generated by R chunks at render time (`echo: false`) so decks
  work offline — never hotlink images.
- Each deck ends by launching a hands-on activity with an explicit file path.

## Before you finish

- Render everything with RStudio's bundled Quarto:
  `PATH="/Applications/RStudio.app/Contents/Resources/app/quarto/bin:$PATH" quarto render`
- Run `Rscript R/verify_simulations.R` if any model code changed.
- Reread one section aloud imagining a tired 15-year-old at 4pm. Cut
  whatever they would skip.
